# Security Compliance — Snyk Vulnerability Monitoring

## Overview

EventHorizon uses [Snyk](https://snyk.io) to continuously scan backend
dependencies for known CVEs and license issues. Scans run on every push/PR
and on a daily schedule so newly-disclosed vulnerabilities are caught without
requiring a code change.

## How it works

```
Push / PR / daily cron
        │
        ▼
.github/workflows/snyk.yml
        │
        ├─ snyk-backend  → scans backend/package.json
        ├─ snyk-root     → scans root package.json
        └─ security-report → generates snyk-report.json artifact
```

Results appear in two places:
- **GitHub Security tab** (Dependabot alerts / Code scanning) via SARIF upload
- **Actions artifacts** — `snyk-report.json` retained for 90 days per run

## Severity policy

| Severity | CI behaviour |
|----------|-------------|
| Critical | Fails the job |
| High     | Fails the job |
| Medium   | Reported, does not fail |
| Low      | Reported, does not fail |

## Setup

1. Create a free Snyk account at <https://app.snyk.io>
2. Copy your **API token** from Account Settings → General
3. Add it as a repository secret: **Settings → Secrets → `SNYK_TOKEN`**

That's all — the workflow activates automatically on the next push.

## Automated dependency updates

`dependabot.yml` is configured to:
- Open daily PRs for `backend/` patch and minor updates (grouped to reduce noise)
- Open weekly PRs for root dependencies and GitHub Actions
- Skip major version bumps (require manual review)

## Compliance report

The `security-report` job produces `snyk-report.json` with this structure:

```json
{
  "generatedAt": "2026-04-29T09:00:00.000Z",
  "project": "backend",
  "compliant": false,
  "summary": {
    "critical": 0,
    "high": 1,
    "medium": 2,
    "low": 3,
    "total": 6
  },
  "vulnerabilities": [
    {
      "id": "SNYK-JS-AXIOS-1234567",
      "title": "Server-Side Request Forgery",
      "severity": "high",
      "packageName": "axios",
      "version": "1.6.0",
      "fixedIn": ["1.7.0"],
      "url": "https://security.snyk.io/vuln/SNYK-JS-AXIOS-1234567"
    }
  ]
}
```

`compliant: true` when there are zero critical or high findings.

The report is generated by `backend/src/utils/securityReport.js` and can be
consumed by any downstream tooling (dashboards, Slack alerts, etc.).

## Running locally

```bash
# Install Snyk CLI
npm install -g snyk

# Authenticate
snyk auth

# Test backend dependencies
cd backend && snyk test --severity-threshold=high

# Generate JSON report
snyk test --json > snyk-report.json
node -e "
  const r = require('./backend/src/utils/securityReport');
  const raw = require('./snyk-report.json');
  console.log(JSON.stringify(r.buildReport(raw, { project: 'backend' }), null, 2));
"
```
